위의 4개의 파일은 시간값에 대한 오류수정이 되지 않은 상태에서 출력한 것이므로
시간이 모두 0으로 나온다. 하지만 보고서에서 기록 하였듯이 최종 코드에는
시간값을 tstamp 변수가 아닌 jiffie 값으로 찍어내며, 시간이 잘 표기된다. 

1.start 

커널 모듈을 올리고, 휴대폰bc:72:b1:d0:2:71 과 desktop20:16:d8:87:9d:89 으로 부터
번갈아 가며 패킷이 온다. 

2.server(odroid) app 의 desktop 버튼을 누르면, 20:16:d8:87:9d:89에 해당하는 패킷은 
"packet is consumed!!" 가 출력되며, 패킷이 들어오지 않는다. 콘솔에는 그와같이 기록되고
app의 경우에는 데이터가 들어오지 않는다. (보고서 참조) 이 때에도 bc:72:b1:d0:2:71에
해당하는 패킷은 잘 통과가 되는 것을 알 수 있다.

3.desktop 버튼을 한번 더 클릭하여 필터링을 끈 후에 otherdev 버튼을 눌러서 휴대폰
bc:72:b1:d0:2:71에 대한 필터링을 시작한다.  20:16:d8:87:9d:89는 잘 들어오나 휴대폰 mac
주소는 consume 된다. 마찬가지로 app에는 데이터가 등장하지 않는다.
